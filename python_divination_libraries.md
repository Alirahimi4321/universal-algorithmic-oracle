> تاریخ تهیه: ۲۰۲۶-۰۶-۱۷ | جستجو در PyPI، GitHub و وب

---

## قانون کلی که اول باید بدونی

- **خیلی‌هاشون وجود ندارن.** از کل لیست شما، حدود **۴۰٪ سیستم‌ها هیچ کتابخانه اختصاصی پایتون ندارن** — به‌خصوص بخش‌های ایرانی/اسلامی، بابلی/مصری، آفریقایی، آندی، مایا/آزتک. این یعنی اگه می‌خوای کدنویسی کنی، یا باید **خودت بسازی** یا از **ابزارهای عمومی** (نجوم، تقویم، متن‌کاوی) به‌عنوان پایه استفاده کنی.
- **جوامع فعال پایتون** بیشتر حول این سیستم‌ها شکل گرفتن: **هندو (Jyotish)**، **چینی (BaZi/Zi Wei/I-Ching)**، **غربی (Tarot/Runes/Numerology)** و **هلسنیستی-اخترشناسی (Hellenistic)**.
- **مثقال دقیق**: سیستم‌هایی مثل «اومن»، «پراشنا»، «شاکونا»، «کائوری‌مانسی» در حد پایان‌نامه و کد پراکنده هستن، نه کتابخانه.

## پیشنهاد عملی من

اگه می‌خوای یه پروژه بسازی، بسته به اهدافت:

1. **اگه می‌خوای سریع نتیجه بگیری** → با `kerykeion` + `lunar-python` + `PyJHora` + `riimut` + `tarot-card-meanings` + `numerology` یه API فال چندفرهنگی بساز.
2. **اگه می‌خوای مال خودت باشه و پُر از چیز جدید باشه** → رمل یا جفر بنویس، بازار خالیه.
3. **اگه می‌خوای آکادمیک کار کنی** → تزولکین + تونالپوآلی + آسترولوژی بابلی، پتانسیل مقاله دارن.

## پ.ن.: آسترولوژی و فارسی

برای نجوم ایرانی (ستاره‌های ایرانی، بُروج فارسی) باید شخصی‌سازی کنی چون کتابخانه آماده نیست. جدیدترین ریپازیتوری مرتبط: `katelouie/stellium` و `northtara/jyotishganit` (هردو ۲۰۲۵-۲۰۲۶ فعال). هم `Ehsan Khazeni` (متخصص نجوم ایرانی) ویدئوهایی دارد که مرجع خوبی است.

اگه بخوای، می‌تونم برات یه ماژول پایه‌ای **رمل اسلامی** یا **جفر** بسازم که الان اصلاً وجود نداره — فقط بگو کدوم رو ترجیح می‌دی.
